## Features

### Build system

- Config-driven pipeline: all behavior defined in YAML, no Python changes needed to add a language or swap source fonts
- Three-layer config merge: font identity, build settings, and logging are loaded independently and deep-merged at runtime
- `extends` inheritance â€” profile YAMLs only specify what changes from their parent
- Environment variable override for the paths file (`GORU_PATHS_FILE`)

### Profiles

Four built-in profiles:

| Profile | Half-width | Full-width |
|---|---|---|
| `mono` | 1024 | 2048 |
| `mono-c` | 870 | 1740 |
| `absolute-mono` | 1024 | 1024 |
| `absolute-mono-c` | 870 | 870 |

Multi-profile builds via `-p mono mono-c â€¦` run sequentially with per-profile headers and a final summary.

### Language processing (FontForge)

- English, Korean, Japanese supported out of the box
- Per-language processing order, timeout, vertical shift, and overlap removal controlled entirely by config
- Glyph classification by Unicode range: `no_center`, `no_scale`, `halfwidth`, `fullwidth`, and threshold-based auto-detection
- Cross-language overlap removal (`remove_overlaps_with`) ensures English glyphs take precedence over CJK variants at the same code point
- Jinja2-rendered PE scripts â€” templates receive full language config, no hardcoded language names in Python

### Parallelism

- Styles (`regular`, `bold`, `italic`, `bold_italic`) processed in parallel via `ProcessPoolExecutor`
- Worker count configurable with `-w` (default: 4)
- `--sequential` flag for single-style debugging

### Post-processing

- `fonttools` writes `post.isFixedPitch`, `OS/2.panose.bProportion`, and `OS/2.xAvgCharWidth` to every generated TTF
- Ensures correct monospace recognition in VS Code, Windows Terminal, and other applications

### Logging

- Hierarchical terminal output with ANSI 256-color support
- Separate console and file log levels
- Optional Rich-powered parallel progress display (requires `rich`)
- Timestamped log files written to `logs/`

### Temp directory management

- Timestamped working directory created per build run under `temp/`
- Automatic cleanup on normal exit and SIGINT/SIGTERM
- `output.save_temp_files: true` preserves generated PE scripts to `backups/` for inspection

### Validation

- Config completeness checked before any file is created
- Source font existence verified per style per language
- Missing fields reported with exact YAML paths
